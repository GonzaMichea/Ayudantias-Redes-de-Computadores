1. Fundamentos del Direccionamiento

Dirección MAC vs. Dirección IP

Todo dispositivo en una red necesita una identificación única. Existen varios tipos :

  • Dirección MAC: Es un identificador físico de 6 bytes (48 bits) asignado a la tarjeta de red. Sirve solamente para identificar un aparato en la red local .
  • Dirección IP: Sirve para identificar de forma única el computador en Internet . Tiene un tamaño de 32 bits (4 bytes) en IPv4, usualmente escrita en decimal separados por puntos .

Anatomía y Netmask

El Netmask (Máscara de red) determina cuál es la parte que identifica la red en la dirección IP de la parte que identifica el host . Para obtener la red, se hace un AND lógico entre la IP y la Netmask .

Además, existe la Dirección de Broadcast, que sirve para mandar un mensaje a todos los computadores conectados a una red . Se calcula poniendo con el valor 1 todos los bits de host .

2. Clases de IP y Redes Privadas

Clasificación Histórica de IP

Clase A
  • Rango: 0.0.0.0 hasta 127.255.255.255 .
  • Netmask: 255.0.0.0 = /8 .
  • Capacidad: 16 Millones de computadores por red .
Clase B
  • Rango: 128.0.0.0 hasta 191.255.255.255 .
  • Netmask: 255.255.0.0 = /16 .
  • Capacidad: 65.534 computadores por red (16 Bits) .
Clase C
  • Rango: 192.0.0.0 hasta 223.255.255.255 .
  • Netmask: 255.255.255.0 = /24 .
  • Capacidad: 254 computadores por red (8 Bits) .
Clases Especiales (D y E)
  • Clase D: Direcciones usadas para el multicast . En este contexto no hay noción de redes .
  • Clase E: Direcciones reservadas para uso experimental, desarrollos futuros e investigación .

Direcciones Privadas y NAT

Las direcciones privadas son usadas para tener un gran número de hosts, pero con una pequeña cantidad de direcciones públicas . No son ruteables en Internet .

Tipo / Bloque Inicio Fin
1 Clase A (/8 prefix) 10.0.0.0 10.255.255.255
16 Clases B (/12 prefix) 172.16.0.0 172.31.255.255
256 Clases C (/16 prefix) 192.168.0.0 192.168.255.255
Traducción de Red (NAT)

Para salir a Internet, el router emplea NAT. Se usa una dirección pública con un número de puerto dinámico . Esto permite hacer un puente (Proxy) entre la red privada (empresa, universidad) y la red externa .

4. DNS: El Sistema de Nombres de Dominio

Resolución de Nombres

Usar direcciones IP es necesario para identificar máquinas, pero es difícil de usar para los usuarios . El Domain Name Server (DNS) asocia un nombre único y fácil de recordar a cada computador mediante una solución jerárquica .

Herramientas Diagnósticas
  • nslookup (Name Server Lookup) : Utilidad básica para consultar IPs de dominios.
  • dig : Herramienta más avanzada para interrogar servidores DNS y ver registros específicos (A, NS, CNAME).
  • Resolución Inversa (Reverse Resolution): Permite encontrar el nombre desde una IP utilizando el dominio especial in-addr.arpa .

Administración y Jerarquía

Los dominios en Internet forman un árbol. La raíz (Root Domain) y los dominios de Top Level son manejados directamente por el ICANN (Internet Corporation for Assigned Names and Numbers) .

RIRs (Regional Internet Registries): Manejan el primer sub-dominio. Por ejemplo, LACNIC cubre América Latina y el Caribe .

Dentro de un país, existen entidades locales (como NIC Chile para el dominio .cl) .
